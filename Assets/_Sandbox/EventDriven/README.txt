独自のライフサイクルを定義した抽象クラスを定義。
そのほかのクラスは上記のクラスを継承することでライフサイクルの様々なタイミングで処理を実装することができる。

長所 :
- イベント実行順がクリアになる
- ヒエラルキー上での疎結合を達成できる部分が多くなり、ヒエラルキーの柔軟性が上がる
- ~Manager的なクラスが膨大にならない
- Componentの疎結合性が高くなる

短所 :
- 様々な箇所でイベント関数が呼ばれるが、その順番を管理できない

改善点 :
- イベントの通知範囲を制限する

不明点 :
- Behaviorを継承したオブジェクトがかなり増えてきたとき実行速度が遅くなるのが怖い。。
